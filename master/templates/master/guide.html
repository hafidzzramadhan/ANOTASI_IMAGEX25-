Apa itu Master?

Master = role tertinggi di platform ini. Lu bertanggung jawab:

  • Bikin job anotasi (project)
  • Upload dataset image
  • Assign tim annotator + reviewer
  • Monitor progress kerjaan
  • Putus dispute (issue solving) kalo annotator vs reviewer beda pendapat
  • Export dataset hasil buat training AI
  • Kelola anggota tim (invite, ubah role, deactivate)
2

Workflow Lengkap

Alur kerja standar di platform ini:

1
Master bikin job → upload image → assign annotator + reviewer
2
Annotator kerja anotasi (bbox/polygon) → submit untuk review
3
Reviewer cek hasil → approve atau reject + bikin issue
4a
Kalo ada issue: Annotator rework → submit ulang
4b
Kalo annotator gak setuju: eskalasi ke Master → putus
5
Master export dataset → kirim ke klien / train AI
3

Bikin Job Baru

  1. Klik menu Job Settings di sidebar
  2. Klik tombol + Bikin Job Baru
  3. Isi form:
    • Title: nama project (e.g. "Tag Mobil di Jalan")
    • Description: deskripsi singkat
    • Segmentation type: semantic / instance / panoptic
    • Shape type: bounding box / polygon
    • Priority: low / medium / high / urgent
    • Start date + End date: timeline project
  4. Klik Save → job dibikin, status default not_assign
  5. Buka detail job → klik Upload Image → drag-drop banyak image sekaligus
4

Assign Annotator + Reviewer

  1. Di detail job, klik tombol Assign
  2. Pilih Annotator dari dropdown (cuma user role annotator yg muncul)
  3. Pilih Reviewer
  4. Save → status job otomatis jadi in_progress
  5. Annotator + reviewer dapet notifikasi otomatis
💡 Tip: kalo belum ada user annotator/reviewer, daftar dulu di menu Assign Roles → ubah role user dari guest.
5

Monitor Progress

3 cara monitor:

1. Home Dashboard — KPI cards real-time (total job, image, issue per status)
2. Process Validation — per job, lihat detail image: unannotated, annotated, in review, in rework, issue, finished
3. Performance — produktivitas tim (per annotator/reviewer)
6

Issue Solving (Putus Dispute)

Kalo annotator gak setuju sama reject reviewer, mereka bisa eskalasi ke master. Lu jadi hakim final.

  1. Buka menu Issue Solving
  2. Filter tab Active → lihat issue eskalasi (yellow badge)
  3. Klik issue → modal terbuka: image, deskripsi, comment thread
  4. Baca konteks, kasi comment kalo perlu klarifikasi
  5. Pilih keputusan:
    • Side Annotator — annotation diterima, image finished
    • Side Reviewer — annotator harus rework, image jadi in_rework
    • Need Clarification — tunggu info tambahan, status tetap eskalasi
    • Close (override) — tutup paksa
7

Export Dataset

Pas job selesai, download anotasi untuk training AI:

  1. Buka menu Job Settings
  2. Cari job yang udah finish
  3. Klik tombol 📥 Export di card job
  4. Pilih format: JSON (custom) atau COCO (industry standard)
  5. File ke-download otomatis
💡 Buat Klien ML/AI: pake format COCO — langsung kompatibel sama YOLOv8, Detectron2, MMDetection, dll.
8

Kelola User

Di menu Assign Roles:

  • Lihat list semua user + status (active/inactive)
  • Ubah role via dropdown (guest → annotator/reviewer/master)
  • Bulk update — pilih banyak user → apply role sama
  • 🗑 Soft delete — user gak bisa login lagi, tapi data anotasi tetep aman
  • ✓ Activate — aktifin kembali user yg di-deactivate

❓ FAQ

User baru daftar otomatis aktif?

Iya. User signup → langsung bisa login. Tapi role default guest — gak bisa apa-apa sampai master kasih role.

Issue 'open' vs 'eskalasi' bedanya?

open = reviewer reject, annotator belum respon. eskalasi = annotator gak setuju, minta master putuskan.

Kapan job otomatis jadi 'finish'?

Pas semua image status nya finished (reviewer approve semua).

Image yg di-delete bisa di-recover?

Image hard delete (permanent). Hati-hati. Buat user, soft delete (bisa di-reactivate).

Bisa edit job setelah ada anotasi?

Bisa, tapi hati-hati. Ubah shape_type bisa bikin anotasi lama gak valid. Diskusi dulu sama tim.
